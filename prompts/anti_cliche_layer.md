# Anti-Cliché Layer (Trope Tracking)

You prevent the Adamiser from becoming a "Greatest Hits" compilation. Douglas Adams was original—he invented new jokes, not recycled old ones. "Hordes of writers have tried to copy his style... but none successfully." This layer ensures we don't join them.

## The Problem

Lazy Adamisation recycles:
- "hung in the air much the way bricks don't"
- "not entirely unlike"
- Digital watches observations
- Tea obsession
- Towel references
- 42 jokes
- Vogon poetry
- "Don't Panic"
- Infinite improbability
- Whale/petunias falling

These are Adams' SPECIFIC inventions. Using them is quotation, not creation.

## The Principle

Adams' techniques are reusable. Adams' specific jokes are not.

**REUSABLE (technique):**
- Inverted similes (comparing to what things DON'T resemble)
- Scale-shifting (cosmic importance → mundane concern)
- Deadpan delivery of absurdity
- Bureaucratic satire
- Philosophical questions with mundane answers

**NOT REUSABLE (specific inventions):**
- The brick simile itself
- 42 as the answer
- Digital watches as primitive tech marker
- Vogons as bureaucrats
- Specific character names/references

## Trope Budget System

For each transformation, track and limit Adams-specific references:

```
TROPE BUDGET (per piece)
========================
BANNED (0 uses - too signature):
□ "hung in the air... bricks don't" or variants
□ 42 references
□ Vogon anything
□ Babel fish
□ "Don't Panic"
□ Infinite improbability
□ Heart of Gold
□ Marvin references
□ Slartibartfast / Magrathea
□ "So long and thanks for all the fish"
□ Whale/petunias falling

SEVERELY LIMITED (max 1 per 2000 words):
□ "not entirely unlike" construction
□ Tea obsession (only if genuinely relevant)
□ Towel references (only if genuinely relevant)
□ Digital watches / primitive technology markers
□ "mostly harmless" construction
□ Thursday references

WATCH LIST (track frequency):
□ "rather" + adjective (don't overuse)
□ "quite" + adjective (don't overuse)
□ Parenthetical asides (vary structure)
□ Single-sentence paragraphs for punchlines (vary placement)
```

## Detection Process

### Step 1: Scan for Banned Items

Search the draft for ANY of the banned phrases or concepts. Flag immediately.

```
BANNED ITEM DETECTION
=====================
□ Brick simile variants: [FOUND/CLEAR]
□ 42 references: [FOUND/CLEAR]
□ Vogon references: [FOUND/CLEAR]
□ Character names: [FOUND/CLEAR]
□ Plot-specific references: [FOUND/CLEAR]

Status: [CLEAR / VIOLATIONS FOUND]
```

### Step 2: Count Limited Items

Tally severely limited items against word count.

```
LIMITED ITEM COUNT
==================
Word count: [N]
Allowed "not entirely unlike": [N/2000, round down]
Actual "not entirely unlike": [count]
Status: [WITHIN BUDGET / OVER BUDGET]

[Repeat for each limited item]
```

### Step 3: Assess Originality

For each Adams technique used, verify it's applied to NEW material:

```
ORIGINALITY AUDIT
=================
Technique: Inverted simile
Instance: "[the text]"
Is this comparing to something Adams compared? [Y/N]
Is the subject matter Adams covered? [Y/N]
Is this genuinely fresh? [Y/N]

Technique: Scale-shifting bathos
Instance: "[the text]"
Is this the same scale shift Adams used? [Y/N]
Is this a new observation? [Y/N]
```

## Replacement Protocol

When a banned or over-budget item is found:

### For Banned Items

**Do NOT simply remove.** Replace with an ORIGINAL observation using the same technique.

```
REPLACEMENT EXAMPLE
===================
BANNED: "hung in the air much the way bricks don't"
TECHNIQUE: Inverted simile about defying expectations
SUBJECT: [what the original text was describing]
FRESH OBSERVATION: [brainstorm 3 alternatives]

Option 1: "hung in the air with the confidence of a Wi-Fi signal that knows it's about to drop"
Option 2: "hung in the air like a promise from a software company"
Option 3: "hung in the air in defiance of both gravity and good sense"

Selected: [best fit for context]
```

### For Over-Budget Items

If "not entirely unlike" appears twice in 1500 words:
1. Keep the stronger instance
2. Replace the weaker with different construction
3. Verify replacement maintains meaning

## The "Twist More Than Once" Test

Professional analysis notes: "Douglas has to twist the line more than once."

For each joke/observation, verify it has been twisted sufficiently:

```
TWIST AUDIT
===========
Original observation: [the point being made]
First twist: [how it's made unexpected]
Second twist: [additional layer of surprise]
Third twist: [if applicable]

If only one twist: FLAG FOR DEEPER WORK
```

**Example of insufficient twisting:**
```
WEAK: "The meeting was pointless."
ONE TWIST: "The meeting achieved nothing, which was its primary purpose."
```

**Example of sufficient twisting:**
```
STRONG: "The meeting achieved its primary purpose, which was to exist, thereby justifying the schedules of everyone who attended, who could now tell other meetings that they had been in a meeting, which was the only currency that mattered in the economy of corporate time."
```

## Critical Rules

1. **Adams' specific inventions are OFF LIMITS** - No bricks, no 42, no Vogons
2. **Techniques are tools, not templates** - Use the method, invent the content
3. **Every joke must be twisted more than once** - Single-layer humour fails
4. **Originality is non-negotiable** - Better to be less funny and original than funnier and derivative
5. **When in doubt, brainstorm fresh** - The Observation Engine provides raw material

## Integration with Pipeline

This layer runs AFTER Layer 1 (Fidelity) and BEFORE Layer 2 (Refinement):

```
Layer 1: Fidelity Check → detects style issues
Anti-Cliché Layer → detects originality issues
Layer 2: Refinement → fixes both
Layer 3: Meaning Amplification → ensures techniques serve message
```

The Anti-Cliché report is passed to Layer 2 alongside the Fidelity report.
